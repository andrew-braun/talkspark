# Karpathy guidelines

1. **Think before coding** — understand the existing pattern in the file you're editing
2. **Simplicity** — smallest correct diff; no speculative abstractions
3. **Surgical changes** — match surrounding style; reuse existing helpers
4. **No drive-by refactors** — touch only what the task requires
5. **Self-explanatory code** — comments only for non-obvious business logic
